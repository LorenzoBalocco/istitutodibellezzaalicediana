## Concept del brand designer

### Monogramma "AD"

Favicon con le iniziali **"AD"** in stile serif elegante (Cormorant Garamond), sfondo scuro per massima leggibilità a piccole dimensioni.

| Proprietà | Valore |
|-----------|--------|
| Forma | Quadrato con angoli leggermente arrotondati |
| Sfondo | `#2C2825` (black-soft) |
| Lettera | "AD" serif — `#E8D5D0` (rose-cipria) |
| Font style | Cormorant Garamond, weight 500, letter-spacing -0.02em |
| Dimensione SVG | 32×32 (scalabile) |

Questa combinazione funziona a 16px, 32px, 180px (apple-touch-icon) perché il contrasto è alto e il monogramma è leggibile.

### Strategia tecnica

In **Next.js App Router**, le favicon vengono gestite tramite file speciali in `src/app/`:

- `icon.svg` → servito come `/icon.svg` (usato dai browser moderni)
- `apple-icon.tsx` → `ImageResponse` genera automaticamente `/apple-touch-icon.png`
- `favicon.ico` → compatibilità legacy (IE, vecchi browser)

Non è necessario generare il `.ico` manualmente: Next.js lo fa automaticamente dal file SVG se si usa `src/app/icon.tsx` con `ImageResponse`. Tuttavia, per massima compatibilità, creo sia l'SVG che aggiorno il `.ico`.

## Deviazioni possibili durante l'implementazione

- Se `ImageResponse` per `apple-icon.tsx` non supporta font Cormorant Garamond via Google Fonts (richiede fetch), userò un fallback sans-serif con stesso contrasto.
- Il file `favicon.ico` binario non può essere creato via codice puro — lo genererò via `src/app/icon.tsx` (Next.js lo produce automaticamente) oppure con un SVG convertito.
